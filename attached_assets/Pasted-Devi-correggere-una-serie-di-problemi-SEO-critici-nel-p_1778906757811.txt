## 1. GOOGLE SITE VERIFICATION — index.html

## 2. PAGINA 404 — src/pages/not-found.tsx
Sostituisci tutto il contenuto del componente con questo, senza modificare nulla:
- Aggiungi in cima: import { SeoHead } from "@/components/seo-head";
- Nel return, prima di qualsiasi altro elemento, aggiungi:
  <SeoHead title="Pagina non trovata | prevai" description="La pagina che cerchi non esiste." canonical="" noIndex={true} />
- Sostituisci il testo "Did you forget to add the page to the router?" con: "La pagina che stai cercando non esiste o è stata spostata."
- Sostituisci il titolo "404 Page Not Found" con: "404 — Pagina non trovata"
Se il componente SeoHead non accetta una prop "noIndex", aggiungila come prop booleana e usala per renderizzare <meta name="robots" content="noindex, nofollow" /> nell'head.

## 6. LINK INTERNI IN CITY-LANDING — src/pages/seo/city-landing.tsx
In questo file, ogni tag <a href="..."> che punta a un URL interno del sito (che inizia con / e non con http) deve essere sostituito con il componente <Link href="..."> di wouter.
Il tag <a> è già importato in wouter come Link. Verifica che l'import { Link } from "wouter" sia presente in cima al file.
Non toccare i link esterni (quelli che iniziano con https:// o http://).

## 7. DATE MODIFIED NEL BLOG — src/pages/blog/[slug].tsx
Nel JSON-LD dell'articolo, trova questa riga:
dateModified: article.publishedAt,
Sostituiscila con:
dateModified: article.updatedAt ?? article.publishedAt,
Se il tipo dell'articolo (nel file di dati del blog) non ha il campo updatedAt, aggiungilo come campo opzionale: updatedAt?: string;
Non aggiungere questo campo agli articoli esistenti — lascialo undefined di default.
